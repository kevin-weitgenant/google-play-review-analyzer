# Plan 6.1 — Replace httpx with AsyncGroq SDK

## Problem

After switching to `openai/gpt-oss-20b` with strict `json_schema`, the Groq API returns errors. The raw `httpx` calls don't handle the request/response the same way the official Groq SDK does (auth headers, request formatting, etc.). The SDK is battle-tested and handles edge cases that manual HTTP calls miss.

## What Changes

### 1. Install the Groq SDK

```bash
uv add groq
```

### 2. Rewrite `backend/app/services/groq_service.py`

**Remove:**
- `httpx` dependency — no more manual HTTP calls
- `tenacity` retry decorator — SDK has built-in retries with exponential backoff
- Manual `GROQ_CHAT_URL` constant
- Manual `Authorization` header construction
- `response.raise_for_status()`, `response.json()`, fragile `json.loads(raw)` parsing
- `client: httpx.AsyncClient` parameter from both function signatures

**Add:**
- `from groq import AsyncGroq` — async client for FastAPI compatibility
- Module-level `AsyncGroq(max_retries=3)` instance (reads `GROQ_API_KEY` from `.env` automatically)
- Direct `await client.chat.completions.create(...)` calls

**Keep:**
- `SENTIMENT_JSON_SCHEMA` and `PRIORITY_JSON_SCHEMA` constants — pass as `response_format`
- Prompt assembly logic (HEADER + optional instructions + FOOTER)
- `.get()` validation and enum fallback as defense-in-depth
- Logging

**New function signatures:**
```python
async def analyze_sentiment(content: str, instructions: str = "") -> str:
async def analyze_priority(content: str, instructions: str = "") -> str:
```

No more `client` parameter — the SDK manages its own HTTP client internally.

### 3. Update `backend/app/routers/reviews.py`

**Remove:**
- `import httpx`
- `async with httpx.AsyncClient() as client:` block
- Passing `client` to `analyze_sentiment()` / `analyze_priority()`

**Simplify to:**
```python
tasks = []
for review in reviews:
    tasks.append(analyze_sentiment(review.content, request.sentiment_instructions))
    tasks.append(analyze_priority(review.content, request.priority_instructions))
results = await asyncio.gather(*tasks, return_exceptions=True)
```

### 4. Cleanup dependencies

After the rewrite, `httpx` and `tenacity` may no longer be needed. Verify if anything else uses them before removing from `pyproject.toml`.

## Why the SDK Handles Retries Better

From the Groq README:

> Certain errors are automatically retried 2 times by default, with a short exponential backoff. Connection errors, 408 Request Timeout, 409 Conflict, 429 Rate Limit, and >=500 Internal errors are all retried by default.

We set `max_retries=3` to match our current tenacity config. The SDK also covers **429 Rate Limit** errors which our current tenacity setup doesn't explicitly handle.
