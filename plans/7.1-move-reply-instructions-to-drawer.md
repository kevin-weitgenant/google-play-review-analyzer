# Plan 7.1: Move Reply Instructions into the Review Drawer

## Context

The reply instructions textarea was on the first screen (input screen) inside the collapsible "Configuracoes da IA" section. This was bad UX — the user had to think about reply style before even seeing any reviews. We moved it into the review drawer where the user actually generates replies, so they set instructions and click "Gerar resposta com IA" in one place.

---

## Changes

### Frontend (`design-kanban.tsx`)

1. **Removed** "Como gerar respostas?" textarea from the input screen's advanced settings
2. **Removed** `replyInstructions` from the `AIConfig` interface (no longer a global config)
3. **Added** `replyDraft` local state (`useState("")`) — scoped to the current drawer session
4. **Replaced** the "Nenhuma resposta gerada ainda" placeholder with a new layout:
   - A textarea labeled "Instrucoes para a resposta (opcional)" bound to `replyDraft`
   - Placeholder text: "Ex: Sempre inicie pelo nome do usuario. Seja empatico e direto..."
   - The "Gerar resposta com IA" button below it
5. **Updated** `handleGenerateResponse` to accept an `instructions` parameter
6. **Updated** both call sites (generate + regenerate) to pass `replyDraft`
7. **Updated** `generateReply()` to accept and send `instructions` to the backend

### Backend (`app/schemas/api.py`)

Added `reply_instructions` field to `GenerateReplyRequest`:

```python
reply_instructions: str = Field(
    default="",
    description="Optional extra instructions for reply generation.",
)
```

### Backend (`app/routers/reviews.py`)

Added logging for the new field:

```python
logger.info("reply_instructions: %s", request.reply_instructions)
```

### Orval — API client regeneration

Ran `npx orval` to regenerate the client. `GenerateReplyRequest` now includes:

```ts
reply_instructions?: string;
```

---

## UX Flow (Before → After)

### Before
1. User opens app → expands "Configuracoes da IA"
2. Types reply instructions in a textarea (before seeing any reviews)
3. Pastes URL → clicks "Analisar agora"
4. Opens a review drawer → clicks "Gerar resposta com IA"
5. Instructions from step 2 were used (but disconnected from context)

### After
1. User pastes URL → clicks "Analisar agora" (only priority + sentiment config on this screen)
2. Opens a review drawer → sees the review text
3. Optionally types reply instructions right there ("Be empathetic, mention the team was notified...")
4. Clicks "Gerar resposta com IA" → instructions + review sent to backend together
5. Reply appears in the textarea, user can edit and mark as done
