# Plan 7.2: Groq-Powered Reply Generation

## Goal

Replace the placeholder reply in `POST /api/reviews/generate-reply` with an actual Groq API call, following the same HEADER + INSTRUCTIONS + FOOTER pattern used by sentiment and priority.

---

## 1) Prompt Structure

Same 3-part pattern as sentiment/priority:

| Part | Who controls it | What it does |
|------|----------------|--------------|
| **HEADER** | Backend (hardcoded) | Describes the task: you are a developer support agent, write a professional reply to the review |
| **INSTRUCTIONS** | Frontend (optional) | User's custom rules from the drawer textarea |
| **FOOTER** | Backend (hardcoded) | The review context (user name + review text) + JSON format instruction |

### 1a) Default prompt (no custom instructions)

What Groq receives when the user leaves the textarea empty:

```
You are a professional app developer responding to Google Play Store reviews.
Write a concise, helpful reply to the review below.

Rules:
- Start by greeting the user by their first name.
- Acknowledge their feedback before offering any explanation.
- Be empathetic and direct. Avoid generic responses.
- If it's a bug or technical issue, mention that the team has been notified.
- If it's positive feedback, thank them sincerely.
- Keep the response under 3 sentences.
- Write in the same language as the review.

Respond with a JSON object: {"reply": "your response here"}

Reviewer: João Silva
Review:
App crashes every time I try to log in. This has been happening since the last update.
```

## 2) Implementation

### File: `backend/app/schemas/api.py`

Add the prompt constants:

```python
REPLY_PROMPT_HEADER = (
    "You are a professional app developer responding to Google Play Store reviews. "
    "Write a concise, helpful reply to the review below.\n\n"
    "Rules:\n"
    "- Start by greeting the user by their first name.\n"
    "- Acknowledge their feedback before offering any explanation.\n"
    "- Be empathetic and direct. Avoid generic responses.\n"
    "- If it's a bug or technical issue, mention that the team has been notified.\n"
    "- If it's positive feedback, thank them sincerely.\n"
    "- Keep the response under 3 sentences.\n"
    "- Write in the same language as the review."
)

REPLY_PROMPT_FOOTER = (
    'Respond with a JSON object: {{"reply": "your response here"}}\n\n'
    "Reviewer: {user_name}\n"
    "Review:\n"
    "{content}"
)

DEFAULT_REPLY_INSTRUCTIONS = ""
```

### File: `backend/app/services/groq_service.py`

Add a JSON schema for the reply and a new async function:

```python
REPLY_JSON_SCHEMA = ResponseFormatResponseFormatJsonSchema(
    type="json_schema",
    json_schema={
        "name": "reply",
        "strict": True,
        "schema": {
            "type": "object",
            "properties": {
                "reply": {
                    "type": "string",
                }
            },
            "required": ["reply"],
            "additionalProperties": False,
        },
    },
)


async def generate_reply(
    user_name: str,
    content: str,
    instructions: str = "",
) -> str:
    """Generate a developer reply to a review. Returns the reply text."""
    parts = [REPLY_PROMPT_HEADER]
    if instructions:
        parts.append(instructions)
    parts.append(REPLY_PROMPT_FOOTER.format(
        user_name=user_name,
        content=content,
    ))
    user_message = "\n\n".join(parts)

    response = await client.chat.completions.create(
        model=settings.groq_model,
        messages=[{"role": "user", "content": user_message}],
        temperature=0.7,        # higher than sentiment/priority — we want creative replies
        max_tokens=300,
        response_format=REPLY_JSON_SCHEMA,
    )

    raw = response.choices[0].message.content or "{}"
    result = json.loads(raw)
    return result.get("reply", "")
```

Note: `temperature=0.7` instead of `0` — reply generation benefits from some creativity, unlike classification which should be deterministic.

## 4) Key Design Decisions

| Decision | Reasoning |
|----------|-----------|
| **temperature=0.7** | Replies need some creativity; 0 is too robotic |
| **max_tokens=300** | Replies should be concise (3 sentences), no need for 500 |
| **JSON structured output** | Same pattern as sentiment/priority — consistent, parseable |
| **No star rating in prompt** | Keeps it simple — just the name and the review |
| **"Write in the same language as the review"** | Auto-detects language — works for Portuguese, English, Spanish, etc. |
| **Custom instructions in the middle** | Same injection pattern as sentiment/priority — user can't break the format |
