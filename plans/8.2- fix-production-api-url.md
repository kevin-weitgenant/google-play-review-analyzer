# Plan 8.2: Fix production API URL

## Problem
The frontend on Render is trying to hit `http://localhost:8000` instead of the production URL. This is because `.env.production` has `VITE_API_URL=` (empty), which falls back to `http://localhost:8000`.

Since FastAPI serves both the API and the frontend on the same origin, the fix is to set `VITE_API_URL` to an empty string so the browser uses **relative URLs** (same origin).

## Change

| File | Action |
|------|--------|
| `frontend/.env.production` | Set `VITE_API_URL` to empty string with no fallback to localhost |

### Current code in `axios-instance.ts`:
```ts
baseURL: import.meta.env.VITE_API_URL || "http://localhost:8000"
```
Problem: when `VITE_API_URL` is empty, JS treats it as falsy and falls back to `localhost`.

### Fix:
```ts
baseURL: import.meta.env.VITE_API_URL ?? "http://localhost:8000"
```
- `??` (nullish coalescing) only falls back on `null`/`undefined`, NOT on empty string `""`
- So `VITE_API_URL=` (empty) → baseURL = `""` → relative URLs → same origin → works on Render
- Dev with no `.env` → `undefined` → falls back to `http://localhost:8000` → works locally

No hardcoded Render URL needed — this works on any deployment.
